N-body simulation dataset

  • All datasets are in binary and compressed using bzip2 due to its size. Please verify the checksum on download.
  • The file name encodes snapshot number and particle type. For example, file cosmo25cmb.768g.00043_gas.bin contains all gas particles in snapshot 43.

Schema

  • The data format is described in the IASDS 2009 paper. All numbers are stored in little endian and all fields except iOrder are 4 byte floating point type (i.e., float4 type). iOrder is 8 byte signed-integer type (i.e., int8 type).
  • Gas: iOrder, mass, position x, y, z, velocity x, y, z, phi, rho, temp, hsmooth, metals
  • Dark matter: iOrder, mass, position x, y, z, velocity x, y, z, phi, eps
  • Star: iOrder, mass, position x, y, z, velocity x, y, z, phi, metals, tform, eps

Data Files

  • DBTEST (21 GB)
  • COSMO50 (10 GB)
  • COSMO25 (60 GB)
  • MD5 checksum

Convert to CSV

For your convenience, we provide a python script that can convert the binary format to CSV format.

  • bin2csv.py -- requires Python 2.5 or higher
